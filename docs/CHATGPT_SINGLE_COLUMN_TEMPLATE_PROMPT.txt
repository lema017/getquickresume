Eres un experto en generar plantillas de CV como Web Components (JavaScript vanilla). Necesito que generes un **template de CV de una sola columna** que se integre en GetQuickResume. ## Formato técnico - **Un solo archivo .js** que se ejecute en un IIFE: (function() { 'use strict'; ... })(); - **Custom Element**: extiende HTMLElement, se registra con customElements.define. El nombre del tag debe ser exactamente gqr-resume-XXXX (ej: gqr-resume-forest) y el nombre de la clase GQRResumeXxxx (PascalCase del sufijo). - **Shadow DOM**: usa attachShadow({ mode: 'open' }), todo el HTML y CSS van dentro del shadow root. - **Cabecera del archivo** (comentario JSDoc al inicio):
/**
   * name: gqr-resume-XXXX
   * description: "Una frase que describa el estilo visual del template (header, secciones, colores, tipografía)."
   */
- **Datos**: el componente recibe un objeto data por propiedad (setter/getter). No uses atributos para el contenido. - **Idioma**: atributo language ('en' | 'es'). Incluir i18n con claves para es y en (profile, experience, education, projects, certifications, languages, achievements, skills, present, y levelMap para basic/intermediate/advanced/native). - **Utilidades obligatorias**: safeStr(v), safeArr(v), escapeHtml(t), y una función para formatear rango de fechas que use "Present"/"Presente" si isCurrent/isCompleted === false en educación. Formato de fecha corto (ej: "Jan 2024" o "Ene 2024" según idioma). ## Orden estricto de secciones (single-column) Las secciones deben aparecer en este orden. Si una sección no tiene datos, no la renderices. 1. **Header** (nombre, profesión, contacto) 2. **Profile** (resumen) 3. **Skills** 4. **Work Experience** 5. **Projects** 6. **Achievements** 7. **Education** 8. **Certifications** 9. **Languages** ## Estructura del objeto data Usa exactamente estos campos (el backend inyecta este formato): - **Header**: firstName, lastName, profession, email, phone, country, linkedin - **Profile**: summary - **Skills**: skillsRaw (array de strings), toolsRaw (array de strings). Combinar ambos sin duplicados para la sección de habilidades. - **Experience**: experience — array de objetos con: id, title, company, location, startDate, endDate, isCurrent, achievements (array), responsibilities (array). Bullets = achievements + responsibilities. - **Projects**: projects — array con id, name, description, technologies (array), url (opcional). - **Achievements**: achievements — array con id, title, description, year (opcional). - **Education**: education — array con id, degree, field, institution, gpa (opcional), startDate, endDate, isCompleted. - **Certifications**: certifications — array con id, name, issuer, date (opcional). - **Languages**: languages — array con id, name, level (basic | intermediate | advanced | native). Mostrar nivel traducido con levelMap. ## Atributos para paginación - Cada sección debe estar envuelta en un contenedor con data-section="header", data-section="contact", data-section="profile", data-section="skills", data-section="experience", data-section="projects", data-section="achievements", data-section="education", data-section="certifications", data-section="languages" según corresponda. - Cada ítem dentro de listas (experiencia, educación, proyectos, etc.) debe tener data-entry-id="..." con el id del elemento (o un identificador único). En skills puedes usar data-entry-id="skill-0", skill-1, etc. ## Diseño y CSS - **Página**: contenedor principal con clase tipo .page: width: 210mm, min-height: 297mm, height: auto, overflow: visible, fondo blanco (o el que definas). Padding coherente (ej: 32–44px). - **Impresión**: incluir al final de los estilos: @media print { .page { width: 210mm; min-height: 297mm; } } - **Colores en impresión**: usar -webkit-print-color-adjust: exact; print-color-adjust: exact; en :host si hay fondos o acentos de color. - **Tipografía**: sin imágenes ni fotos. Usar fuentes seguras o Google Fonts si quieres un estilo más distintivo. - **Responsive**: el CV es para A4; no hace falta diseño responsive para móvil. ## Variaciones de estilo que puedes usar (ejemplos) - **Header**: centrado o alineado a la izquierda; barra de color (navy, verde, gris) con nombre en blanco; solo nombre + línea sutil; nombre + profesión en la misma línea. - **Contacto**: en línea bajo el nombre; en pills/badges; con iconos (✉ ☎ ⚲ 🔗); en una fila flexible con wrap. - **Títulos de sección**: mayúsculas con línea inferior; cursiva serif con subrayado corto; barra lateral de color; iconos circulares; líneas decorativas a los lados del texto. - **Experiencia / Educación**: fechas a la izquierda (columna fija ~120px) y contenido a la derecha; o fecha a la derecha en la misma línea que el título; bullets con viñeta o guión. - **Skills**: lista con bullets; grid de 2 o 3 columnas; badges/pills con bordes redondeados; con o sin descripción. - **Idiomas**: texto "Nombre — Nivel"; o dots de nivel (●○○○○) con leyenda basic/intermediate/advanced/native. - **Paleta**: claro (fondo blanco, acentos discretos), oscuro (fondo navy/charcoal, texto claro), o un acento único (rojo, azul, verde, dorado) sobre blanco. ## Checklist antes de entregar - [ ] IIFE y 'use strict' - [ ] Comentario con name y description al inicio - [ ] Clase que extiende HTMLElement con data (get/set), connectedCallback, attributeChangedCallback para language, y método render() que escribe en this.shadowRoot.innerHTML - [ ] getLanguage() que lee atributo language o data.language (fallback 'en') - [ ] i18n (es/en) para títulos de sección y "Present"/"Presente" y levelMap - [ ] Orden de secciones: 1→9 como arriba - [ ] data-section en cada bloque de sección - [ ] data-entry-id en cada ítem de lista - [ ] .page con 210mm × min-height 297mm, overflow visible - [ ] @media print y print-color-adjust si hay color - [ ] customElements.define('gqr-resume-XXXX', GQRResumeXxxx) solo si el elemento no está ya registrado - [ ] Sin imágenes ni fotos Entrega solo el código del archivo .js, listo para guardar como gqr-resume-XXXX.js.

## Después de generar el template: verlo en local

1. **Guardar el .js**  
   Guarda el código que te devuelva ChatGPT en:
   ```
   public/templates/designs/gqr-resume-XXXX.js
   ```
   (Sustituye `XXXX` por el nombre que hayas pedido, ej. `mint`, `forest`.)

2. **Registrarlo en el catálogo**  
   En `src/utils/templateCatalog.ts`, añade una línea al array `TEMPLATE_CATALOG`:
   ```ts
   { id: 'gqr-resume-XXXX', name: 'Nombre para la UI', category: 'free' },
   ```
   - `id`: debe ser exactamente el mismo que el nombre del archivo sin `.js` (y que el tag del custom element en el .js).
   - `name`: el nombre que verás en el selector de plantillas (ej. "Mint", "Forest").
   - La página de selección de plantillas (`TemplateSelectionPage`) y la ruta `/create` leen la lista desde este catálogo; no hace falta tocar ningún otro archivo.

3. **Recargar la app**  
   Con el sitio corriendo en local (ej. `./start-dev.sh` o `npm run dev`), recarga la página. El nuevo template aparecerá en el paso de elegir plantilla, en la vista previa/final y en `/create`.
