# GetQuickResume — Prompt for Two-Column Resume Templates (ChatGPT)

Use this document to generate a **two-column** resume template (sidebar + main column) that is compatible with GetQuickResume: data shape, hydration (pagination), project/experience/education structures, and section order rules.

Copy the sections below into ChatGPT and add your style description at the end.

---

## Role and output

You are an expert at building resume templates as **vanilla JavaScript Web Components**. Generate a **two-column** resume template that integrates with GetQuickResume. Output a single **.js file** only (no HTML/CSS files). The template must consume the app’s `data` object, respect section order, and use the DOM structure required for hydration (pagination).

---

## Technical format

- **Single file**: one `.js` file wrapped in an IIFE: `(function() { 'use strict'; ... })();`
- **Custom Element**: extends `HTMLElement`; tag name exactly `gqr-resume-XXXX` (e.g. `gqr-resume-dock`); class name `GQRResumeXxxx` (PascalCase of the suffix); register with `customElements.define('gqr-resume-XXXX', GQRResumeXxxx)` only if not already defined.
- **Shadow DOM**: use `attachShadow({ mode: 'open' })`; all markup and styles live in the shadow root.
- **Header comment** (JSDoc at top of file):
  ```
  /**
   * name: gqr-resume-XXXX
   * description: "Two-column resume with [brief visual description: sidebar/main style, colors, typography]."
   */
  ```
- **Data**: the component receives a `data` object via a **property** (getter/setter). Do not use attributes for resume content.
- **Language**: support attribute `language` ('en' | 'es'). Implement `getLanguage()` that returns `this.getAttribute('language')` or `this.data?.language` or `'en'`.
- **i18n**: provide an `i18n` object with keys for `es` and `en`: profile, experience, education, projects, certifications, languages, achievements, skills, contact, present. Provide `levelMap` for language levels: basic, intermediate, advanced, native (in both es and en).
- **Helpers**: implement `safeStr(v)`, `safeArr(v)`, `escapeHtml(t)`, and a date-range formatter that uses "Present" / "Presente" when the job or degree is current (`isCurrent` / `isCompleted === false`). Use short dates (e.g. "Jan 2024", "Ene 2024") according to `language`.

## Section order rules (two-column)

Order is **strict**. Only render a section if it has data. Do not change the order.

### Sidebar (narrow column) — only these three sections

All sidebar content must be inside a single container with **class `sidebar`** so the app can detect two-column layout and run pagination correctly.

| Order | Section   | data-section value |
|-------|-----------|--------------------|
| 1     | Contact   | contact            |
| 2     | Languages | languages          |
| 3     | Skills    | skills             |

- **Contact**: email, phone, country, linkedin (any subset present).
- **Languages**: list of languages with name and level (use levelMap for translated level).
- **Skills**: combine `skillsRaw` and `toolsRaw` (if present), dedupe; render as badges. Use `data-entry-id="skill-0"`, `skill-1`, … for each skill so pagination can split skills if needed. **Skills layout**: use a flex-wrap container so badges flow horizontally and wrap to multiple rows (e.g. `display: flex; flex-wrap: wrap; gap: 6px` on the wrapper, and each skill as an inline-block span with `white-space: nowrap`). Do not stack skills vertically (one per row); that wastes horizontal space in the sidebar.

### Main column (wide column) — only these sections

All main content must be inside a single container with **class `main`**.

| Order | Section      | data-section value |
|-------|--------------|--------------------|
| 1     | Header       | header             |
| 2     | Profile      | profile            |
| 3     | Work Experience | experience      |
| 4     | Projects     | projects           |
| 5     | Achievements | achievements       |
| 6     | Education    | education          |
| 7     | Certifications | certifications   |

- **Header**: full name (firstName + lastName), profession. Can be the first block inside `.main`, or a full-width bar above both columns (still use data-section="header").
- **Profile**: summary text.
- **Experience / Education**: each entry must have `data-entry-id` equal to the item’s `id` (e.g. exp.id, edu.id).
- **Projects / Achievements / Certifications**: same — each entry must have `data-entry-id` equal to the item’s `id`.

Do **not** put contact, languages, or skills in the main column if they are already in the sidebar. Do **not** put profile, experience, projects, achievements, education, or certifications in the sidebar. The app’s pagination logic only recognizes sidebar sections as contact, languages, and skills.

## Hydration and pagination requirements

GetQuickResume measures the template’s DOM to distribute content across A4 pages. For two-column layout it:

1. Looks for an element with **class `sidebar`** and an element with **class `main`** inside the root page container.
2. For every element with attribute **`data-section`**, it checks whether that element is inside `.sidebar` or `.main` and assigns it to the sidebar or main column.
3. For list sections (experience, education, projects, achievements, certifications, languages, skills), it uses **`data-entry-id`** on each item to know which entries belong to which page when content overflows.

To be compatible:

- **Root container**: a single element with class **`page`** (e.g. `<div class="page">`), containing:
  - Optionally a full-width **header** block (name + profession) with `data-section="header"`.
  - A single element with class **`sidebar`** containing only the three sidebar sections (contact, languages, skills), in that order, each with the correct `data-section` and `data-entry-id` where applicable.
  - A single element with class **`main`** containing header (if not full-width), profile, experience, projects, achievements, education, certifications, in that order, each with the correct `data-section` and `data-entry-id` on every list item.
- **Section wrappers**: each logical section must be wrapped in an element that has **`data-section="<section>"`** with one of: contact, languages, skills, header, profile, experience, projects, achievements, education, certifications.
- **List items**: every item in experience, education, projects, achievements, certifications, and languages must have **`data-entry-id="<id>"`** set to that item’s `id`. For skills use `data-entry-id="skill-0"`, `skill-1`, etc.

Example structure:

```html
<div class="page">
  <!-- optional: full-width header -->
  <div class="sidebar">
    <div data-section="contact">...</div>
    <div data-section="languages">...</div>
    <div data-section="skills">...</div>
  </div>
  <div class="main">
    <div data-section="header">...</div>
    <div data-section="profile">...</div>
    <div data-section="experience">
      <div data-entry-id="exp-1">...</div>
    </div>
    <div data-section="projects">
      <div data-entry-id="proj-1">...</div>
    </div>
    ...
  </div>
</div>
```

- **Page dimensions**: `.page` should have `width: 210mm`, `min-height: 297mm`, `height: auto`, `overflow: visible`. Include `@media print { .page { width: 210mm; min-height: 297mm; } }`. If the template uses colored backgrounds, set `-webkit-print-color-adjust: exact; print-color-adjust: exact;` on `:host`.

## Layout and styling

- **Sidebar**: typically 30–38% width; often a distinct background color; text and icons sized for a narrow column.
- **Main**: remaining width (e.g. 62–70%); usually white or light background; same A4 and print rules as above.
- **No images or photos** in the template.
- Use system fonts or a single Google Font if desired; keep the template self-contained in one .js file.
